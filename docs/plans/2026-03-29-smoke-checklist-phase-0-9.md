# Mind Map 阶段 0-9 冒烟清单

本清单用于阶段 `0-9` 之后的最小回归确认，覆盖导入导出、富文本、公式、AI、预览、协同六类能力。

## 1. 构建与产物

- `cd app && npm run build`
- `cd app && npm run build:library`
- `cd web && npm run build`

通过标准：

- 命令退出码为 `0`
- 产物落到预期目录
- `app` 和 `web` 首页都能正常打开

## 2. 基础编辑

- 打开编辑页
- 新建根节点子节点
- 删除节点
- 拖拽节点
- 切换布局
- 切换主题
- 切换基础样式

通过标准：

- 不出现空白画布
- 不出现节点无法选中、无法拖动、无法编辑

## 3. 富文本 / 备注 / 公式

- 打开节点备注编辑
- 输入普通 Markdown 文本
- 输入代码块
- 输入数学公式
- 打开备注预览
- 打开富文本编辑
- 打开公式侧栏并插入一条公式

通过标准：

- 备注编辑器能打开
- Markdown 能渲染
- 代码块有高亮
- 公式渲染正常，不出现原始转义文本

## 4. 导入

至少验证以下格式各一例：

- `.smm`
- `.json`
- `.xmind`
- `.md`

通过标准：

- 文件能读取
- 脑图结构不为空
- 节点层级和文本没有明显错乱

## 5. 导出

至少验证以下格式各一例：

- `.smm`
- `.json`
- `.svg`
- `.png`
- `.pdf`
- `.xmind`
- `.txt`

通过标准：

- 可触发导出
- 产物文件可打开
- 文本内容、结构、主题样式没有明显缺失

说明：

- `xlsx` 当前 UI 不是主路径，本轮不作为必须项，但底层链路仍保留。

## 6. AI 能力

- 打开 AI 对话
- 输入一条普通问题
- 输入一条要求生成脑图的提示词
- 将 AI 返回结果渲染为 Markdown

通过标准：

- 请求可以正常发出
- 返回结果可显示
- Markdown 标题、列表、代码块显示正常
- 生成脑图链路不报错

## 7. 图片预览与主题资源

- 在节点中插入图片
- 点击图片预览
- 缩放图片
- 关闭预览器
- 切换扩展主题

通过标准：

- 图片能正常打开和关闭
- 不出现遮罩无法关闭或预览器残留
- 主题切换后画布样式正常

## 8. 协同链

- 启动：`cd simple-mind-map && npm run wsServe`
- 确认日志出现监听成功信息
- 若本地有两个会话，尝试双端连接
- 修改一侧节点文本，确认另一侧同步

通过标准：

- 服务能正常监听
- 不因依赖升级导致启动失败
- 双端同步正常

注意：

- 若出现 `EADDRINUSE`，先检查是否有上次测试残留进程占用 `4444` 端口，不要误判为代码故障。

## 9. 已知非阻塞项

- `simple-mind-map lint` 目前存在历史遗留错误，不作为阶段 `0-9` 通过与否的新增回归判断依据。
- `web build` 的大包体积 warning 已知存在，本轮不处理。
- `web` 对 `--openssl-legacy-provider` 的依赖属于阶段 `10` 处理范围。
