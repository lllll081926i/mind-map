# 编辑页体验优化设计

**日期：** 2026-04-16

## 1. 目标

在不改动编辑页整体布局和视觉结构的前提下，优先提升两类体验：

1. 编辑连续性：让用户更清楚当前文档状态、保存状态、恢复状态，以及切换上下文时的风险。
2. 操作效率：缩短高频动作路径，降低重复搜索、重复保存、重复确认带来的阻力。

本期不引入多标签、多窗口、全新工作流页或复杂编辑器重构，所有优化都基于现有工具栏、搜索面板、状态同步链路和本地文件工作流完成。

## 2. 当前痛点

### 2.1 状态反馈不够完整

当前顶部状态区只能显示“自动保存中 / 已恢复 / 存在未保存变更 / 已保存”，但用户仍然不知道：

- 当前编辑的是哪个本地文件
- 最近一次成功落盘是什么时候
- 当前内容是否只是恢复草稿
- 保存失败后当前处于什么状态

这会导致“功能上可用，心理上不稳”。

### 2.2 离开确认过于笼统

当前返回首页、打开���件、打开最近文件、打开目录、新建文件等动作虽然都有离开确认，但标题和内容完全一致，用户无法快速判断“我这次确认到底会做什么”。

### 2.3 显式保存动作缺位

当前工具栏只有“另存为”，而 `Ctrl+S` 实际已经可以保存当前本地文件。也就是说：

- 系统支持“直接保存”
- 界面没有显式表达“直接保存”

这会让本地文件工作流的认知断裂。

### 2.4 搜索连续性一般

当前搜索面板关闭后会清空输入和结果。对于频繁切换画布和搜索的用户来说，重复打开搜索面板时需要重新输入关键词，效率偏低。

## 3. 本期范围

本期编辑页体验优化收口为 4 个模块：

1. 工具栏状态反馈增强
2. 上下文切换确认收紧
3. 本地文件保存动作补齐
4. 搜索面板连续性增强

## 4. 设计方案

### 4.1 工具栏状态反馈增强

保留现有顶部状态区位置，不新增新的状态栏，只增强已有状态卡片的信息表达。

新增信息：

- 状态副文案：根据当前状态显示更具体的说明
- 当前目标文件摘要：优先显示当前文件名，并通过 `title` 暴露完整路径
- 最近一次成功保存时间：用于“已保存”与“存在未保存变更”场景
- 保存失败状态：在写入失败后显示错误态，直到下次成功保存

状态表达原则：

- 自动保存中：告诉用户“正在写入当前文件”
- 已恢复：明确这是从恢复草稿继续，而不是正式落盘后的状态
- 存在未保存变更：告诉用户当前仍在编辑链路中，若有历史成功落盘则补充最近一次成功保存时间
- 已保存：明确已经写回当前文件，并显示最近成功保存时间
- 保存失败：明确本次写入失败，避免用户误以为已经成功保存
- 无本地文件：明确当前内容仍停留在应用会话中，提示应保存到本地文件

同时去掉打开文件后的常驻通知，避免工具栏顶部状态已经存在时还叠加一个长期不消失的提示层。文件打开成功改为轻提示，恢复草稿保留一次性反馈。

### 4.2 上下文切换确认收紧

保留当前统一的确认弹窗能力，但改为按动作生成标题和文案。

覆盖动作：

- 返回首页
- 打开本地文件
- 打开最近文件
- 打开目录
- 从目录树切换文件
- 新建文件

确认文案要明确说明即将发生的动作，例如：

- “返回首页前，当前未保存内容可能只保留在恢复草稿中”
- “打开其他文件前，当前编辑进度尚未写回原文件”

这样可以减少用户误判，也能降低“同一个弹窗看多了直接确认”的习惯性误操作。

### 4.3 本地文件保存动作补齐

工具栏在现有“另存为”旁边补一个“保存”动作，但只在当前存在本地文件上下文时显示。

行为定义：

- `保存`：写回当前正在编辑的本地文件
- `另存为`：保存为新的本地文件

保留 `Ctrl+S` 作为快捷保存方式，同时让工具栏动作与快捷键含义一致。

本期不新增复杂禁用态逻辑，不引入保存版本管理，也不改变底层本地写入链路。目标只是让界面把现有能力显式表达出来。

### 4.4 搜索面板连续性增强

搜索面板关闭时不再完全丢弃用户输入意图，而是保留最近一次搜索关键词与替换输入草稿。

具体策略：

- 关闭搜索面板时结束当前搜索高亮，但保留最近一次输入草稿
- 再次打开时恢复关键词输入，并自动聚焦
- 如果已有关键词，则直接选中输入框内容，方便用户继续搜索或直接改写
- 仍然在文档切换时清空搜索上下文，避免把旧文档搜索状态串到新文档

这样可以改善高频搜索用户的体验，又不会把旧结果错误地保留到新的文档内容上。

## 6. 测试策略

本期采用“结构断言 + 行为链路断言”的组合测试方式：

- 工具栏测试：验证新增保存入口、状态副文案、失败态、动作化离开确认存在
- 搜索测试：验证搜索草稿恢复、再次打开自动聚焦/选中、切换文档时重置
- 更新现有测试，避免把状态实现细节卡死在某一段字面量写法上

## 7. 验收标准

- 用户能在编辑页顶部明确看懂当前保存状态和当前文件上下文
- 本地文件工作流中，界面同时提供“保存”和“另存为”两类动作
- 未保存内容下执行高风险切换动作时，弹窗会说明具体动作，而不是统一模板
- 关闭搜索面板后再次打开，可以延续最近一次输入，不必完全重输
- 整体界面布局不发生明显改版，仍保持现有工具栏和面板结构
