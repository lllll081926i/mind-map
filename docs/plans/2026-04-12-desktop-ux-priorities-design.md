# 桌面端 UX 优先项设计

**日期：** 2026-04-12

## 1. 目标

在现有“工作区与文档管理”之外，补齐桌面端当前最影响使用感知的一批 UX 优先项，让用户在打开应用、进入编辑、搜索内容、导出结果和处理异常时都能获得更明确、更稳定的反馈。

本期不引入新的文档模型、多窗口、多标签页或云同步能力，重点放在现有首页、编辑页、导出页和设置页的工作流增强。

## 2. 本期范围

本期把此前讨论的体验优先项收口为 5 个可交付模块：

1. 首页工作台升级
2. 编辑器状态反馈与高频入口
3. 搜索与定位增强
4. 导出体验增强
5. 设置页体验提示与风险提示

## 3. 设计原则

- 不新增新的业务中心页，优先复用现有首页、工具栏、导出页、设置页。
- 先解决“用户不知道该做什么”和“系统做了什么用户不知道”这两类问题。
- 所有新增能力尽量使用现有状态源：`workspace meta`、`documentSession`、`exportState`、`Pinia`。
- 风险操作必须给确认；高频操作必须给快捷入口；异步动作必须给状态反馈。

## 4. 模块设计

### 4.1 首页工作台升级

首页继续保留三大核心入口：新建、打开文件、打开文件夹。

在此基础上新增三组内容：

- 工作台信号卡片：显示当前恢复状态、最近文件数量、当前工作目录或最近动作，让用户一眼知道“现在能继续什么”。
- 新手上手区：用 3 步卡片说明“开始编辑、常用快捷键、导出与恢复”，减少第一次使用时的犹豫。
- 体验提示区：集中展示搜索、导出、恢复草稿这三类最常用但容易被忽略的能力。

为了提升体感性能，首页挂载后会提前预热 `workspaceActions` 动态模块，避免第一次点击按钮时再加载。

### 4.2 编辑器状态反馈与高频入口

顶部工具栏新增一个轻量状态区，不替代原有按钮，只补充：

- 当前文档标题
- 当前状态：未连接文件 / 已保存 / 正在自动保存 / 检测到恢复草稿
- 最近一次成功落盘时间或恢复草稿状态

同时新增两个高频入口：

- 搜索
- 快捷键

这些入口使用已有的 `showSearch` 事件和 `shortcutKey` 侧栏，不新增第二套实现。

为了增强容错，以下会切换上下文的动作在存在未保存风险时增加确认：

- 返回首页
- 新建文件
- 打开文件
- 打开最近文件
- 从目录树切换到另一文件

### 4.3 搜索与定位增强

当前搜索已有匹配列表，但缺少“我现在在哪个结果”和“下一步怎么跳”的明确反馈。

本期补充：

- 搜索结果摘要
- 上一个 / 下一个结果切换
- 当前结果高亮
- 搜索操作提示
- 空结果与未开始搜索的说明状态

所有行为基于现有 `simple-mind-map` 搜索插件的事件，不改底层搜索算法。

### 4.4 导出体验增强

导出页已经具备独立预览和格式切换，本期重点增强“可预期性”：

- 记住上次导出选项（格式、留白、透明背景、页脚等）
- 显示当前文档与目标扩展名摘要
- 显示“已记住上次选项”的说明
- 导出成功消息带文件名和扩展名
- 在页面空闲时预热当前导出格式相关插件，降低首次导出等待感

这里只记住导出偏好，不记住文件名，文件名仍然以当前文档名推导，避免误导用户把旧文档名导出到新文件。

### 4.5 设置页体验提示与风险提示

设置页不扩展大量新配置，而是增加说明型内容：

- 工作流建议卡片：提醒恢复草稿、导出中心、快捷键入口
- 体验优化建议卡片：提醒开启工具栏文字、性能模式、AI 配置位置
- 恢复文件卡片保留，并强化“仅清理恢复文件，不影响正式文档”的风险提示

设置页目标不是变成控制台，而是作为“查找说明和确认状态”的稳定入口。

## 5. 测试策略

- 首页布局测试：验证工作台信号卡片、新手步骤、体验提示区存在。
- 工具栏测试：验证状态区、搜索入口、快捷键入口和离开确认逻辑存在。
- 搜索测试：验证结果导航、激活态、摘要文案存在。
- 导出页测试：验证导出偏好记忆、状态摘要、成功反馈文案存在。
- 设置页测试：验证体验提示卡片存在，恢复文件提示文案保留。

## 6. 验收标准

- 首次打开首页时，用户能在一个屏幕内看懂如何开始、如何继续、如何导出。
- 编辑页能明确显示当前保存/恢复状态，并对高风险切换动作给出确认。
- 搜索结果能快速在多个匹配项之间切换并感知当前位置。
- 导出页会记住上次导出选项，并在成功时给出更明确反馈。
- 设置页能向用户解释恢复、导出和快捷键相关能力，而不仅是开关堆叠。
