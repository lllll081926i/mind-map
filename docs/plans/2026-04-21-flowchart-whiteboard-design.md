# 流程图白板式重构设计

**日期：** 2026-04-21

## 1. 目标

当前流程图编辑器已经具备基础文档能力，但整体体验仍明显停留在“功能堆在一起”的阶段：

- 主工作流依赖远处面板，连接节点路径长
- 画布不是主角，工具和说明占据太多注意力
- 顶栏、悬浮面板、节点样式都偏重，白板感不足
- 节点与连线缺少对象级上下文工具，操作不够直接

本期目标是把流程图模式重构为“轻快白板感”：

- 画布成为绝对主角
- 80% 高频操作在节点和连线附近完成
- 右侧面板从主工作流降级为精修区
- 视觉风格参考 `Whimsical + FigJam`
- 连线交互参考 `Miro + draw.io`

你明确补充的两个视觉约束需要强制纳入本期：

- 节点圆角不要太大
- 节点上不要出现多余的类型文字或辅助文字，只保留节点正文

## 2. 设计原则

### 2.1 交互优先级

优先级从高到低如下：

1. 在画布上快速创建和连接节点
2. 快速改字和调整局部结构
3. 在不离开当前上下文的前提下微调属性
4. 导出、模板、AI 等低频动作

### 2.2 视觉原则

- 轻，不厚重
- 清楚，不拥挤
- 默认克制，只在用户需要时显露上下文工具
- 用结构和位置表达操作意图，而不是靠说明文字堆提示

### 2.3 架构原则

- 保留现有文档模型、保存链路、恢复链路和测试体系
- 重构交互层和视觉层，不引入新的大型外部图库
- 拆分 `FlowchartEditor.vue`，避免继续膨胀为超大单文件

## 3. 当前问题归纳

### 3.1 主工作流错误

当前新增节点、连接节点、删除选择、复制粘贴、对齐等动作大量依赖浮动面板。用户需要频繁在“节点”和“右侧面板”之间来回移动注意力，导致：

- 连接动作慢
- 心智负担大
- 画布上的流程搭建不连续

### 3.2 视觉层级错误

当前界面存在以下问题：

- 顶栏按钮密度高，像后台工具条
- 浮动面板过重，边框、阴影、面积都偏大
- 节点像配置卡片，不像白板上的流程块
- 空画布提示和说明文字过多

### 3.3 对象级交互不足

当前节点只有选择、拖拽、双击改字；连线只有选中和属性编辑。缺失以下关键能力：

- 节点周围直接发起连接
- 从节点直接长出下一个节点
- 在连线上插入中间节点
- 连线附近的轻量上下文工具

## 4. 方案选择

本期采用“白板式主工作流 + 轻量上下文工具 + 弱化侧栏”的中等强度重构方案。

不采用仅换皮的轻改方案，因为那只能解决“难看”，不能解决“连接麻烦”。

不采用完全重写交互引擎的重构方案，因为当前仍可复用：

- 现有节点/边数据结构
- 历史记录与持久化链路
- 导出与文件 I/O
- 多选、吸附、对齐、复制粘贴等基础能力

## 5. 用户体验重构

### 5.1 画布主工作流

用户搭建流程图的主要路径调整为：

`选中节点 -> 从连接手柄拖出 -> 连接已有节点或生成新节点 -> 继续生长`

具体规则：

- 单选节点后显示四向连接手柄：上、下、左、右
- 拖拽到已有节点时直接建立连接
- 拖拽到空白处时，按方向创建一个新节点并自动连线
- 方向默认影响新节点初始布局：
  - 右侧：横向流程
  - 下侧：纵向步骤
  - 左侧和上侧：反向补充或回路

### 5.2 节点交互

- 单击：选中节点
- `Shift/Ctrl/Meta + 单击`：追加或取消多选
- 双击：直接编辑节点正文
- 选中时显示连接手柄和极轻的选中描边
- 多选时不显示每个节点的完整手柄，避免画面拥挤

节点正文规则：

- 只显示 `node.text`
- 不显示 `start/process/decision/input/end` 等类型字样
- 节点正文垂直水平居中

### 5.3 连线交互

- 单击连线：选中连线
- 选中后在中点附近显示迷你工具条
- 迷你工具条只保留高频动作：
  - 编辑标签
  - 插入节点
  - 删除连线

插入节点规则：

- 在原连线中点创建新节点
- 原连线断开
- 自动生成两条新连线：`source -> newNode -> target`
- 新节点类型默认 `process`

### 5.4 右侧精修区

右侧面板从主入口降级为次级精修区：

- 默认关闭
- 只有在用户需要改节点类型、尺寸、边文案、网格、模板等细项时使用
- 不再承载“连接节点”“新增下一个节点”这类高频动作

### 5.5 顶栏

顶栏改为轻工具条，保留以下动作：

- 返回
- 保存
- 另存为
- 模板
- AI 生成
- 导入导图
- 导出
- 视口缩放与适应视图

下列内容应从主顶栏降权或移除：

- 冗余说明文字
- 大面积分组块
- 与当前对象强相关的局部动作

### 5.6 空画布

空画布不再使用大段说明卡片，而是改为更轻的起步态：

- 主行动：创建第一个节点
- 次行动：使用模板
- 次行动：AI 生成

说明文案缩到最少，不在空画布上堆功能介绍。

## 6. 视觉设计

### 6.1 整体氛围

参考 `Whimsical + FigJam`，做浅色白板感：

- 背景采用干净的暖灰白
- 网格非常淡，只作为辅助
- 面板和浮层边框更弱、阴影更轻
- 统一单一强调色处理选中态、连接手柄、吸附线和焦点态

### 6.2 节点视觉

- 节点形状保持清爽，不做大圆角胶囊
- `process` 和 `input` 采用小圆角矩形
- `start` 和 `end` 使用较扁的胶囊，但圆角控制在克制范围
- `decision` 继续使用菱形，不额外附加标题或图标
- 节点默认边框细、阴影弱、背景纯净

明确约束：

- 圆角整体向小收敛
- 不在节点顶部额外渲染类型名、提示名或装饰标题

### 6.3 连线视觉

- 默认连线颜色使用中性偏冷灰
- 选中态切换为强调色
- 箭头更明确，但不过度粗重
- 标签默认弱化，仅在有文字或选中时明显显示

### 6.4 悬浮控件视觉

- 连接手柄采用小尺寸圆点或圆角方点
- 连线工具条采用极窄胶囊工具条
- 交互反馈采用短距离淡入和位移，不使用笨重弹跳

## 7. 技术实现方案

### 7.1 组件拆分

当前 `FlowchartEditor.vue` 已经承担过多职责。本期拆分出至少以下子组件或渲染模块：

- `FlowchartCanvas.vue`
  - 负责画布事件、节点/边渲染、选择框、视口层
- `FlowchartNodeLayer.vue`
  - 负责节点渲染、节点选中态、连接手柄
- `FlowchartEdgeLayer.vue`
  - 负责边路径、箭头、标签、中点工具条
- `FlowchartToolbar.vue`
  - 负责轻顶栏
- `FlowchartInspector.vue`
  - 负责次级属性精修区

如果本轮时间不足以完全拆到独立文件，至少也要在当前组件内先把：

- 状态计算
- 画布事件
- 节点动作
- 边动作
- 视口动作

按职责整理成清晰分段，避免继续堆在一个方法区。

### 7.2 新增状态

需要引入以下交互状态：

- `hoveredNodeId`
- `connectorPreview`
- `connectorDragState`
- `edgeToolbarState`
- `inlineTextEditorState`

说明：

- `connectorPreview` 用于渲染连接中的预览线
- `connectorDragState` 记录拖拽起点、方向、吸附目标和空白创建位置
- `edgeToolbarState` 记录当前连线迷你工具条位置

### 7.3 节点连接逻辑

新增统一连接入口，避免分散在多个动作里：

- `beginConnectorDrag(nodeId, direction, pointerEvent)`
- `updateConnectorDrag(pointerEvent)`
- `commitConnectorDrag(pointerEvent)`
- `cancelConnectorDrag()`

提交逻辑：

- 命中已有节点：创建或复用对应边
- 命中空白区域：创建新节点并连线
- 命中自身或无效目标：取消

### 7.4 连线插入节点逻辑

新增统一入口：

- `insertNodeOnEdge(edgeId, type = 'process')`

步骤：

1. 定位边的源节点、目标节点和中点
2. 生成新节点
3. 删除原边
4. 创建两条新边
5. 选中新节点并打开文本编辑

### 7.5 文本编辑逻辑

现有双击编辑逻辑需要提升为真正的行内编辑体验：

- 双击节点后进入轻量文本编辑态
- 编辑控件贴合节点，不额外弹窗
- 回车提交
- `Esc` 取消
- 失焦提交

边标签编辑也采用相同思路，尽量不把用户送到远处面板。

### 7.6 视觉样式重构

重构现有 `FlowchartEditor.vue` 内样式变量，保留主题能力，但统一为白板风格变量：

- `--flowchart-canvas-bg`
- `--flowchart-grid-color`
- `--flowchart-node-bg`
- `--flowchart-node-border`
- `--flowchart-node-shadow`
- `--flowchart-accent`
- `--flowchart-accent-ring`
- `--flowchart-floating-bg`
- `--flowchart-floating-border`
- `--flowchart-floating-shadow`

旧变量中偏厚重、偏深的色值需要整体收敛。

## 9. 测试策略

### 9.1 结构测试

更新现有结构断言，覆盖：

- 节点连接手柄存在
- 连线迷你工具条存在
- 顶栏收敛为轻工具条
- 节点中不再出现额外类型文字
- 样式变量切换为新白板体系

### 9.2 交互行为测试

补充或更新以下场景：

- 从节点手柄拖到空白处会创建并连接新节点
- 从节点手柄拖到已有节点会建立连接
- 重复连接已有边时复用边而不是生成脏重复边
- 在线上插入节点会正确拆分原边
- 双击节点只编辑正文，不引入额外标题文本
- 多选态不会渲染整屏连接手柄

### 9.3 回归测试

保留并复用现有测试覆盖：

- 保存 / 另存为
- 恢复草稿
- 导入导图
- AI 生成
- 导出 SVG / PNG
- 历史记录
- 复制粘贴 / 快捷键

## 10. 验收标准

当以下条件全部满足时，本期可视为完成：

- 用户可在节点附近直接完成大部分连接与新增动作
- 右侧面板不再是高频主工作流入口
- 界面整体呈现轻白板感，而非后台工具台风格
- 节点圆角明显收敛，不再显得臃肿
- 节点上只保留正文，没有多余文字
- 连线支持对象级轻量编辑，不必总去属性面板
- 现有保存、恢复、导出和 AI 链路不回归

## 11. 分阶段实施建议

建议按以下顺序落地，降低回归风险：

1. 先重构视觉层和顶栏层级
2. 再补节点连接手柄和连接预览
3. 再补连线中点工具条和插入节点
4. 最后清理右侧面板职责并收敛说明文案

这样即使中途发现交互细节需要回调，也能保持每一步都处于可运行状态。
